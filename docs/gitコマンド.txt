gitコマンド

1. 準備・設定に関するコマンド
まずは自分のパソコン上でGitを使えるようにしたり、GitHubと連携するためのコマンドです。

git init（リポジトリの初期化）

新しく作ったフォルダを「Gitで管理するフォルダ（ローカルリポジトリ）」に設定します。

git clone [URL]（コピーのダウンロード）

GitHub上にある既存のリポジトリを、自分のパソコンに丸ごとダウンロードします。




2. 日常の開発で「毎日必ず使う」超重要コマンド
GitHub Desktopでボタンを押して行っていた操作の「中身」にあたるコマンドです。

git status（現在の状態を確認）

「どのファイルを書き換えたか」「まだ保存していないファイルはどれか」など、現在の状況を一覧表示します。迷ったらまずこれを打ちます。

git add [ファイル名]（ステージングエリアへ追加）

変更したファイルを「次のコミット（保存）の対象」として登録します。

git add . と打つと、変更されたすべてのファイルを一括で登録できます。（GitHub Desktopの左側でファイルにチェックを入れる操作と同じです）

git commit -m "メッセージ"（コミット・変更の確定）

add した変更にメッセージ（名前）をつけて、ローカルリポジトリにガチッと保存します。

git push origin [ブランチ名]（プッシュ・ネットへ送信）

ローカルでのコミットを、GitHub（リモートリポジトリ）へ送信します。（通常、メインのブランチ名は main または master です）

git pull（プル・最新版をダウンロード）

GitHub側にある最新のコードをダウンロードして、自分のパソコンのコードに合体（同期）させます。




3. 歴史や変更点を確認するコマンド
git log（コミット履歴の確認）

過去に誰が、いつ、どんなメッセージでコミットしたかの履歴を一覧表示します。（GitHub Desktopの「History」タブと同じです）

git diff（コードの差分確認）

ファイルを書き換えた際、具体的にどの行を追加・削除したかを緑と赤の文字で画面に表示します。




4. チーム開発や機能追加で使うコマンド（ブランチ）
現在の本番用コード（main）を汚さないように、別の作業部屋（ブランチ）を作って開発するためのコマンドです。

git branch（ブランチの一覧・確認）

現在どんなブランチがあるか、自分が今どのブランチにいるかを確認します。

git checkout -b [新しいブランチ名]（新しいブランチを作って移動）

本番用から枝分かれした新しい作業部屋を作り、そこに移動します。

git switch [ブランチ名]（ブランチの切り替え）

すでにある別のブランチ（部屋）に移動します。
